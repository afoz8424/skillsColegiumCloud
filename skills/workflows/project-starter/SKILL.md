# Skill de Ejecución de Proyectos (Project Starter)

Enfocado en la creación de estructura inicial, boilerplate y base para nuevos proyectos o módulos grandes dentro del workspace.

## Flujo de Trabajo

### 1. Definición del Proyecto
- Entiende el alcance del nuevo proyecto y los requisitos técnicos.
- Identifica tecnologías, dependencias y la estructura de carpetas necesaria.

### 2. Planificación de Estructura
- Crea un diagrama o lista de los archivos base que deben generarse.
- Define convenciones de nombres, rutas (`routes/`), controladores y modelos.

### 3. Scaffolding y Boilerplate
- Usa comandos `php artisan` (si aplica) para generar la base del proyecto.
- Configura las migraciones de base de datos iniciales.
- Establece la estructura de archivos en el directorio `/projects/` o el habitual.

### 4. Implementación Base
- Escribe el código inicial de los componentes críticos.
- Asegura que el proyecto pueda ser ejecutado o desplegado para pruebas tempranas.

### 5. Documentación Inicial
- Genera un archivo README o `ARCHITECTURE.md` para el nuevo proyecto.
- Describe cómo arrancar, probar y desplegar la solución.
